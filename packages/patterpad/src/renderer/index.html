    Localisation

    The project's languages. The default is the source language every string is authored in; the others are localisation targets.

    Drives the lock-aware vs merge-based write path and the project's VCS config files (vcs-setup.md, ignore rules). Switching regenerates those config files for the new system.

    Project-global @patter properties: game state your conditions and effects read and write.

    World properties: the values your game engine owns and your story reads (referenced as @world), plus the inputs the coverage test feeds them.

    The master cast. A line's speaker must match a cast member's name.

    Estimating replaces a still-guesswork scene's line count with a guess in the production report, so a stub scene (a couple of placeholder lines) doesn't read as nearly finished. The estimate is shared across the characters in its placeholder lines. When off, reports show pure actuals.

    Closed captions describe non-spoken sounds inside dialogue. A game can turn captions off at runtime; Patterplay then removes every cue between these delimiters (and the surrounding space) from a line: “Oh dear. [sigh] What now?” → “Oh dear. What now?”. Lines spoken by the caption character are removed entirely. Captions don't affect audio playback.

    The pair that wraps a caption cue inside a line; the default is [ … ]. With captions off, everything between them (and the surrounding space) is removed. Avoid ( as the start: a ( at the start of a line opens a performance direction, so a caption there won't work. Use [ or another token.

    Custom fields the host game reads off each node type. Define them here; edit each node's values in the inspector.
