## Patterpad: the editor

**Patterpad is the main app**: the desktop editor where you write. It is
self-contained; nothing else to install.

<Downloads kind="patterpad" />

:::caution[Windows SmartScreen]
The Windows installer is not code-signed yet, so SmartScreen shows a warning on first
run. Choose **More info → Run anyway**. (macOS builds are signed and notarized.)
:::

### Something to open

New to Patter? Grab the **Interactive Tour** - a small, playable project that teaches the
format by playing it. Unzip it, open the `tour.patter` folder in Patterpad (double-click on
macOS, or **File → Open** on Windows / Linux), and press **Play from Start**.

<Downloads kind="tour" />

## Patterplay: the runtimes

The plugin that plays your story inside your game. Pick the engine you ship on - the
four runtimes are the same version and [tested to match](/play/overview/):

<Downloads kind="plugins" />

The JS runtime is also on npm as `@patterkit/runtime` (plus the optional
`@patterkit/play-helpers`), or from a CDN:

```html
<script src="https://unpkg.com/@patterkit/runtime/dist/patterplay.min.js"></script>
```

See [The Engine API](/play/engine/) and the per-engine guides to get started.

## The CLI

`patter` validates, compiles, plays, and localises projects from the terminal: handy
for CI and automation. One self-contained file per platform, **no Node.js required**:

<Downloads kind="cli" />

Or from npm, if you already have Node:

```sh
npm install -g @patterkit/cli
patter validate ./my-project.patter
```

See the [CLI reference](/cli/) for every command.
